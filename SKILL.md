> **本次更新内容（v44 → v45）**：
>
> **能力融合（来源：罗盘·行业研究引擎）**：
>
> ### 🆕 新增「权力分析框架」
> 产业链拆解每层新增「话语权⭐」维度。基于8项量化指标（毛利率弹性、应收/应付账款差、客户集中度、转换成本等）综合评分，从替代难度深化到"谁在产业链里说了算"。产业链末尾新增「话语权分布总览」汇总表。

>
> ### 🆕 新增「多报告交叉对比」（柔性规则）
> 第二层侦察完成后、第三层裁断之前，新增独立子步骤。搜到2份+不同来源独立报告时，强制对比框架差异、视角矛盾、增量信息。仅1份则标注"仅基于XX一份报告"。0份跳过。
>
> ### 🆕 新增「金字塔原则标题」
> 全报告标题改为判断句，结论先行。禁止功能性标题（如"产业链拆解"→"上游材料环节话语权最强，模块组装利润最薄"）。
>
> ### 🆕 新增「政策权重强调」
> PEST中的政治部分强制第一顺位，强调政策影响但不限定篇幅占比。连锁效应分析中政策变化作为第一个强制展开的触发变量。
>
> ### 🆕 新增「5类行业视角权重表」（内置参考）
> 行业类型路由完成后，查询5类权重表（消费/内容、科技/制造、平台/交易、服务/人力、政策/监管），命中则按视角权重分配搜索精力+推荐信息源，不命中走通用策略。
>
> ### 🆕 新增「HTML可视化输出模板」
> 复用罗盘风格HTML模板渲染报告（卡片式公司清单、三色风险标注、来源标签），与.md双格式并存。标准侦察及以上模式均输出.html。

> **升级了什么**：
>

> - 不通过 → 退回修正 → 重验 → 通过才放行进入铁律2
> - 核心原则：**不通过，不准动笔**
>
> **🔴 铁律3升级为最终闸门**：
> - 验证粒度从"铁律级别"细化到"每条铁律的每条子项"
> - 快速判断模式底线：铁律0 + 铁律A + 铁律B，三条不能少
> - 标准侦察及以上：全部铁律（0/1/2/4/A/B）逐条检验
> - 判断标准从"我做了这个步骤"改为"执行结果在报告中有体现"
> - 退回策略从"整体重来"改为"只退回到未达标子项对应的执行步骤"
> - 新增检验信号：交付前自问"报告读完后，读者能感受到'这个行业既在爆发也在流血'的张力吗？"
> - 辩证性不是靠固定章节实现的，而是靠铁律3的逐条验证贯穿全文

### 铁律0：前置三检自检（环境就绪检查）

> 任何分析执行前，必须首先检查执行环境是否就绪，避免因环境问题导致输出空框架或低分评测。本铁律是所有后续分析的前提条件，不可跳过。

在任何分析执行之前，技能自动执行三道强制检查：

| 检查项 | 检查内容 | 失败时的报错信息 |

### 铁律1：先想后搜（强制底层流程）

每个任务开始前，强制按以下顺序执行，不能跳过、不能合并：

1. 强制看一遍用户提出的问题
2. 对问题进行强制深入分析，看用户到底想问什么
3. 强制先看一遍全流程
4. 强制去思考我应该做什么
5. 强制思考我应该搜什么
6. 强制想我如何安排篇幅
7. 强制思考我能为用户做什么
8. 遇到极度模糊的需求时，强制触发询问先行，强制追问用户需求——追问到一般性需求显现就行，不需要极致细节

### 铁律3：强制铁律验证（最终闸门）——【v46升级：逐条细粒度+双层闸门】

> **位置**：铁律2（多模块独立执行）+ 铁律4（搜索兜底）+ 铁律A（多元视角）+ 铁律B（连锁效应）全部执行完成后。**这是交付前的最后一道闸门。**

**核心原则**：铁律3不是一条独立的执行铁律，而是**验证机制本身**。它不参与自验——其他铁律全部逐条✅后，铁律3的使命自然完成。

**验证粒度弹性**：按执行模式分级——

| 模式 | 验证范围 | 说明 |
|------|---------|------|
| **快速判断** | 铁律0 + 铁律A + 铁律B | 三条底线不能少。快速判断输出简短，铁律A轻量标准：关键判断≥2个视角而非3个；铁律B轻量标准：关键变量标注连锁方向即可，无需完整展开三层连锁 |
| **标准侦察及以上** | 全部铁律（0/1/2/4/A/B） | 每条铁律的每条子项逐条检验 |

**完整验证清单**（标准侦察及以上）：

| 铁律 | 子项数 | 验证内容 |
|------|--------|---------|
| 铁律0 | 3项 | 技能加载 / 沙箱环境 / 联网搜索 |
| 铁律1 | 8项 | 看问题→深入分析→看流程→想做什么→想搜什么→想篇幅→能为用户做什么→模糊追问 |
| 铁律2 | 5+1模块 | 类型路由 / 第一层判断 / 第二层侦察 / 独立专项 / 第三层裁断 / ∞深化 |
| 铁律4 | 3级策略 | 多策略重试→合理推断→诚实标注缺口 |
| 铁律A | 每条核心判断 | 每条核心判断≥3个视角验证 |
| 铁律B | 每个关键变量 | 每个关键数据点/变化信号展开连锁效应 |

**逐条验证执行流程**：

**第一步：逐条提取** —— 翻开每条铁律，提取出该铁律的全部子项要求

**第二步：逐条问"做到了吗"** —— 对每条子项，根据报告内容判断是否执行到位
- 判断依据：不是"我做了这个步骤"，而是"执行结果在报告中有体现"
- 示例：铁律A检查的不是"我有没有写多元视角章节"，而是"报告中的每一条关键判断，是不是真的从至少3个视角交叉验证了"

**第三步：标记状态** —— 每条子项标记✅（达标）或❌（未达标），❌记录具体缺了什么

**第四步：退回修正** —— 所有❌的子项，不是整体退回，而是**只退回到该子项对应的执行步骤**，补充、修正后重新检验，不牵涉其他已✅的子项

**第五步：二次检验** —— 修正后的子项重新检验，通过后标记✅

**第六步：最终放行** —— 全部子项✅后才能交付报告。输出 `【最终闸门】铁律全检通过 ✅ 交付`

**检验信号**：交付前强制回答一个问题——"这份报告读完之后，读者能清晰感受到'这个行业既在爆发也在流血'的张力吗？"如果答案是"偏正面/偏负面"而非"两者都有穿透力"，说明铁律未执行到位，退回。

**禁止行为**：
- 因为"大部分通过了"就放行不通过的子项 ❌
- 把"整体偏正面"当作笼统的修改方向而不逐条定位到具体子项 ❌
- 验证时只看自己"做了什么步骤"，不看"结果在报告中有没有体现" ❌
- 对于不能够明确的部分不标注、不告知用户 ❌
- 快速判断模式跳过铁律0、铁律A、铁律B中的任何一条 ❌

---

### 铁律4：搜索兜底与缺口处理

搜索是分析的基础，但搜索结果并非每次都能满足需求。当搜索数据不足时，按以下三级策略处理，**禁止直接留白或跳过该维度**：

**第一级：多策略重试**
- 换关键词：用户口语→学术术语→英文检索，至少尝试3组不同方向的关键词
- 换引擎：通用搜索→学术搜索→垂直领域搜索
- 换时间范围：扩大时间跨度或缩小到更精确的时段
- 组合搜索：用已有数据点反推的关联信息作为搜索线索

**第二级：基于行业常理的合理推断（仅在第一级全部失败后触发）**
- 输出格式：`「{推断内容}（基于行业常理的合理推断，置信度低，无直接数据支撑，建议核实）」`
- 推断必须有逻辑链可查：因为XX（已知事实）+ 行业通常XX（一般规律）→ 所以XX
- 禁止在没有事实锚点的情况下凭空推断
- 单次分析中，推断占比不得超过该维度内容的30%

**第三级：诚实标注缺口**
- 明确写出数据缺口：`「关键数据缺口：{具体数据项}，未在公开信息中找到可信来源，以下分析跳过此维度」`
- 标注缺口对整体结论的影响：此缺口是否影响核心判断，是局部缺失还是致命盲区
- 缺口累计超过3个维度时，在报告开头额外加一句整体盲区提醒

**执行顺序**：必须按一级→二级→三级顺序执行，不能跳过第一级直接输出推断。每个模块独立执行此流程——A模块搜索失败不影响B模块照常产出。

## 行业类型路由（Step 0）

进入分析前，先判断行业类型，决定第二层走哪条分支：

| 类型 | 特征 | 典型行业 | 第二层重点 |
|------|------|---------|-----------|

## 两条分析铁律（贯穿全流程）

### 铁律A：多元视角

同一个行业，不同立场看到的东西完全不同。**每个关键结论必须从至少3个视角分别论证**：

| 视角 | 看什么 | 关切点 |
|------|--------|--------|
| **投资者** | 回报率、风险收益比、估值合理性 | 买什么、什么价位买、什么信号卖 |
| **产业经营者** | 竞争优势、护城河、战略卡位 | 如何构建壁垒、如何定价、如何绑定客户 |
| **技术演进者** | 技术路线胜率、迭代节奏、替代风险 | 哪条路线赢、什么时候切换、被颠覆的概率 |
| **政策监管者** | 产业安全、自主可控、合规风险 | 哪些环节卡脖子、国产替代进度、管制方向 |
| **下游客户** | 供给稳定性、性价比、切换成本 | 够不够用、贵不贵、能不能换 |

**执行方式**：每当得出一个关键判断（如"1.6T光模块是明年主线"），立即追问——投资者怎么看？经营者怎么看？技术演进者怎么看？如果三个视角得出的结论一致，判断置信度高；如果矛盾，矛盾本身就是最有价值的信息。

**输出格式**：
```
🎯 关键判断：{判断内容}
├── 投资者视角：XXX（置信度XX%）
├── 经营者视角：XXX（置信度XX%）
├── 技术演进视角：XXX（置信度XX%）
└── 视角冲突：XXX（如有矛盾，说明为什么，这往往是最有价值的信息）
```

### 铁律B：连锁效应分析

**数据不是拿来列的，是拿来说明"一个变量动了，整条链怎么跟着动"。**

每个关键数据点/变化信号，必须展开连锁效应：

**框架**：
```
🔔 触发事件：{变量变化}
│
├── 第一层直接影响：XXX
│   ├── 第二层连锁：XXX
│   │   └── 第三层扩散：XXX
│   └── 第二层连锁：XXX
└── 第一层直接影响：XXX
    └── 第二层连锁：XXX

⬆️ 反向传导：下游变化如何向上游传导
💰 利益重分配：谁受益、谁受损、谁被淘汰
⏰ 时间差：从触发到各层响应的时间差（决定了投资/决策的领先/滞后）
```

**举例**（光通讯行业）：
```
🔔 触发事件：InP衬底供给紧缺（全球仅3家供应商，产能扩张周期18-24个月）
│
├── 第一层：EML芯片产能受限 → 1.6T光模块交付周期拉长
│   ├── 第二层：有自研芯片能力的模块厂（中际旭创/新易盛）份额提升
│   │   └── 第三层：无芯片能力的模块厂被迫外购芯片 → 成本上升 → 毛利压缩 → 退出高端市场
│   └── 第二层：客户转向LPO方案（省EML） → LPO路线获得意外加速
└── 第一层：InP衬底涨价 → EML芯片涨价
    └── 第二层：光模块ASP提升 → 短期利好营收，但客户开始评估替代方案

⬆️ 反向传导：下游AI资本开支缩减 → 光模块订单砍单 → 芯片厂产能闲置 → 衬底厂被迫降价
💰 利益重分配：衬底厂（短期涨价的受益者）→ 自研芯片模块厂（份额集中）→ 外购芯片模块厂（被挤压）
⏰ 时间差：衬底扩产18-24个月 > 模块扩产6-12个月 → 缺口持续时间长，利好已有库存/长协的公司
```

**执行要求**：
- 产业链拆解中，每个瓶颈环节必须写连锁效应
- 技术路线交锋中，每条路线的胜出必须写连锁效应（路线A赢了，谁受益谁受损）
- 景气度分析中，需求/供给变化必须写连锁效应
- 裁断层中，核心风险必须写连锁效应

**判断标准**：如果一个数据点只是"XX行业规模XX亿"而没有说明它动了会怎样，这个数据就是死的，必须激活它。

# 第一层：行业判断框架

> 源自肖璟《如何快速了解一个行业》产业生命周期框架

## Step 1：行业定位

界定研究范围——横向（细分行业边界）和纵向（产业链覆盖环节）。

参考分类标准：申万行业分类 / GICS / 证监会行业分类 / 国标。

**输出**：行业名称、分类口径、横向边界、纵向边界、颗粒度说明。

## Step 2：产业生命周期定位

用渗透率划分：

| 阶段 | 渗透率 | 研究重点 |
|------|--------|---------|
| 导入期 | <15% | 可行性：商业模式能否跑通 |
| 成长期 | 15-40% | 规模性：天花板多高，增速多快 |
| 成熟期 | >40% | 防守性+盈利性：护城河+竞争格局+利润分配 |
| 衰退期 | 替代品出现 | 替代品识别 |

判断方法：搜索渗透率数据 → 头部公司营收增速验证 → 融资热度交叉验证。

## Step 3：四维分析

根据生命周期阶段重点不同，但四个维度都要覆盖：

### 可行性（导入期重点）
- 需求验证：时间对标+空间对标
- 商业模式画布（9维度）
- 销售可行性+利润可行性

### 规模性（成长期重点）
- TAM测算（自上而下/自下而上/对标法/渗透率法）
- 3-5年CAGR
- 增速是加速还是减速

### 防守性（成熟期重点）
- 护城河五类：网络效应/转换成本/成本优势/无形资产/高效规模
- 第二增长曲线
- 周期化可能

### 盈利性（成熟期重点）
- 产业链利润分配：哪个环节最赚钱
- 竞争格局：CR3/CR5/CR10
- 波特五力

**输出**：四维评分表（★☆评分+核心判断+置信度）

## Step 4：估值特征

| 生命周期 | 估值方法 |
|---------|---------|
| 导入期 | VC评估法（概率加权） |
| 成长期 | PEG/PS |
| 成熟期-稳定 | DCF/PE/PB |
| 成熟期-周期化 | 周期股估值（PE反着看） |
| 衰退期 | 清算价值/股息率 |

## Step 5：PEST

政治/经济/社会文化/技术——只写与该行业直接相关的，不凑数。

### ⚠️ 政策权重强调

**政治（P）部分强制第一顺位，强调政策影响但不限定篇幅占比。** 中国各行业普遍深度受政策影响，除非该行业明确不受政策驱动（如部分纯消费行业），否则政治分析须作为PEST的核心：

- **产业政策**：国家/省级鼓励类还是限制类、补贴/税收/准入政策
- **监管方向**：合规要求趋严还是放松、数据安全/环保/反垄断
- **地缘影响**：出口管制/技术封锁/国产替代进度
- **十四五/十五五等规划**：纳入什么、排除什么、目标量化

> **连锁效应联动**：政策变化作为铁律B（连锁效应分析）中第一个强制展开的触发变量。任何政策信号出现，必须展开"这个政策动了，产业链怎么跟着动"。

**经济/社会文化/技术**各占剩余篇幅，同样只写与该行业直接相关的，不凑数。

## Step 6：景气度跟踪

量价指标/融资指标/情绪指标/库存指标——只列该行业真正可跟踪的指标，写清楚在哪看。



# 第二层A：产业链拆解（通用框架）

> 核心概念：**产业链是两条反向链交汇的闭环。**
>
> **逆向链（需求拉动）**：用户需求→分销商/渠道→制造商→原材料/供应商——需求从下游向上游传导，反向拉动生产。
>
> **正向链（供给驱动）**：原材料→加工/制造→分销/渠道→最终用户——供给从上游向下游流转，推动全链条运转。
>
> 两条链在中间交汇，形成一个闭环。**分析一个行业，就是走通这两条链**——谁在拉动（需求从哪里来、沿着什么路径向上传导、传导过程中的损耗和时滞），谁在推动（原材料从哪里来、经过哪些加工组装、每一步的技术壁垒和利润分配），以及两条链交汇处的卡点和博弈。
>
> **不是每个行业都有固定层数，每一层的名字也不一样。** 分析者需要先自行解析该行业的产业链具体由哪些环节构成（不同行业的分销层可能是经销商/代理商/门店/电商平台/社群/企业级客户），而不是套模板。

## 通用分析思维

拆解产业链，沿两条链分别追问：

### 逆向链（需求拉动）

| 追问链 | 要回答什么 |
|--------|-----------|
| 1️⃣ 最终需求是谁？ | 最终用户/应用场景、需求强度、增长趋势 |
| 2️⃣ 需求通过什么渠道传导？ | 分销商/经销商/电商/渠道层级、传导效率、库存周期 |
| 3️⃣ 谁来满足这个需求？ | 制造商/服务商格局、产能利用率、订单前置期 |
| 4️⃣ 需求倒逼了什么上游供给？ | 原材料/元器件被拉动、供不应求的信号如何向上传导 |

### 正向链（供给驱动）

| 追问链 | 要回答什么 |
|--------|-----------|
| 5️⃣ 原材料从哪里来？ | 基础原料/元器件/服务的上游供给格局 |
| 6️⃣ 经过什么加工？ | 生产工艺、产能、技术壁垒、成本结构 |
| 7️⃣ 怎么集成/组装/组合？ | 组件集成、系统组装、服务编排 |
| 8️⃣ 通过什么渠道到用户？ | 物流、分销、仓储、库存管理 |
| 9️⃣ 最终交付给谁？ | 与逆向链1️⃣交汇——确认需求端和供给端是否匹配 |

每个行业沿这两条链走一遍，画出自己的产业链图谱。**两条链交汇处往往是卡点和利润集中点。**

## 通用分析维度

对每个产业链环节，问清楚这7个问题：

1. **做什么**：这个环节的核心功能（加工/组装/流通/服务）
2. **全球Top3玩家+份额**：覆盖全球，不能只看中国
3. **中国玩家+进度**：谁在做、做到什么水平
4. **技术/资源壁垒**：为什么难做
5. **替代难度**：⭐-⭐⭐⭐⭐⭐评级，写清卡点
6. **话语权**：⭐-⭐⭐⭐⭐⭐评级，基于罗盘8项指标（毛利率弹性、应收/应付账款差、客户集中度、转换成本等），上市公司财报直接算，非上市定性标注
7. **关键数据**：市场规模、增速、毛利率、价格趋势

## 行业示例：硬科技

硬科技行业的产业链通常可细化为 **材料→芯片→器件→模块/组件→系统/设备→基础设施/运营** 六层，每层套用上述6个维度逐层拆解：

### 1. 上游材料层
- **做什么**：这个行业的基础材料是什么（衬底/晶圆/特种气体/稀土/预制棒等）
- **全球Top3玩家+份额**
- **中国玩家+进度**
- **技术壁垒**：纯度/一致性/良率/设备依赖
- **替代难度**：⭐-⭐⭐⭐⭐⭐
- **话语权**：⭐-⭐⭐⭐⭐⭐（基于毛利率弹性、应收/应付账款差、客户集中度、转换成本等8项指标综合评分）
- **关键数据**：市场规模、增速、毛利率、价格趋势

### 2. 芯片层
- **做什么**：核心芯片类型（激光器芯片/调制器/DSP/控制芯片等）
- **全球Top3玩家+份额**
- **中国玩家+进度**
- **技术壁垒**：制程/设计/工艺/专利
- **替代难度**：⭐-⭐⭐⭐⭐⭐

## 模块一：品牌与商品分析

> 五段递进，从市场扫描到趋势总评，专门分析"这个品牌/商品为什么行/不行"。

## 1.1 市场扫描

- 品类市场规模、增速、渗透率
- 竞争格局：TOP10品牌+份额（国内外）
- 市场集中度（CR5/CR10，上升还是下降）
- 价格带分布：高端/中端/大众/白牌各层占比

## 1.2 品牌辩证

- 品牌定位：每个头部品牌的差异化锚点是什么
- 品牌心智：用户提到这个品类首先想到谁，为什么
- 品牌溢价能力：同品类同规格，能卖贵多少
- 品牌老化/焕新：老牌在衰退还是转型，新牌在崛起还是昙花

## 1.3：消费者心理需求

- 用户购买的核心动机：功能需求/情绪需求/社交需求/身份需求
- 决策路径：从注意到购买的完整链路，关键决策节点
- 情感驱动：什么情感（安全感/归属感/炫耀/怀旧/焦虑）在驱动消费
- 用户分层：重度用户/轻度用户/潜在用户的差异化需求

## 1.4：商品爆发分析

- 爆款商品特征：价格/卖点/包装/渠道/时间节点
- 渠道匹配：什么商品在什么渠道容易爆（抖音适合冲动型，小红书适合种草型）
- 传播规律：怎么火的（KOL引爆/事件营销/口碑裂变/平台推荐）
- 复购与留存：是一次性爆发还是可持续

## 1.5：趋势总评

- 品类生命阶段：导入/成长/成熟/衰退
- 未来1-3年趋势：品类会扩张还是收缩，高端化还是性价比
- 机会点：用户未被满足的需求/品类空白/渠道红利
- 风险点：价格战/渠道依赖/政策风险/审美疲劳

## 第二阶段：深化分析

在第一阶段产出的结论基础上，追加五步深化分析。**每步结论必须与第一阶段及前后步骤交叉验证。**

### 第一步：背景分析（因→果）

回答：**什么因导致了今天的局面？**

第一阶段已经得出了"当前是什么状况"的判断。现在往前追溯——这些状况不是凭空出现的，它们的根在哪里。

- **关键归因**：现状中的每一个核心判断（产业链瓶颈/竞争格局/技术路线），追溯它是怎么形成的——历史关键节点、政策、技术突破、资本潮汐
- **长期驱动因素**：结构性力量（人口结构、代际更替、资本周期、地缘格局）如何在过去5-10年塑造了这个行业
- **之前的预判偏差**：上一轮市场共识为什么错了（如果存在），错在哪

**输出要求**：每一个归因必须与第一阶段结论能对上——"因为背景中的XX，所以现状中的XX成立"。对不上的，要么背景分析有误，要么现状判断需要调整。

### 第二步：现状深化

回答：**当前在什么位置？**

基于第一阶段已有结论，但每个结论都要用背景分析的结果做交叉验证：

- 产业链瓶颈真的是今天才有的吗？它在背景里已经埋了多久？
- 竞争格局的变化，在背景中有无先兆？
- 第一阶段中置信度低于60%的判断，经过背景验证后是否更清晰了？

**输出要求**：标出每个核心结论在背景验证后的置信度变化——↑（背景支持，置信度提升）/ ↓（背景矛盾，需重新审视）/ →（背景无法验证，维持原判）。

### 第三步：推演预测

回答：**因为现在的状况，未来会走向哪里？**

- **内在趋势推演**：现状中的结构性力量（已由背景分析确认）会自然导向什么方向
- **触发点识别**：什么变量可能打破当前趋势（政策转向/技术突破/黑天鹅）
- **情景分析**：乐观/中性/悲观三种路径，以及各路径的置信度
- **时间框架**：未来1年/3年/5年的关键节点

**输出要求**：每个推演结论标注——基于背景中的XX趋势 + 现状中的XX判断 → 推论出XX。不标注因果链的推演无效。

### 第四步：正反面案例交叉分析

在第一阶段的竞争格局分析和第二阶段的推演结论基础上，选取同行业/同赛道/同路线中的**反面失败案例**和**成功典型案例**做对比交叉分析。

> 反面案例不是点缀，是验证结论的关键手段。如果反面案例的失败原因指向了与主流判断相反的结论，这个矛盾本身就是最有价值的信息。

#### 反面案例选取标准

- 同一个行业/赛道里失败的典型企业
- 同一技术路线下被淘汰的参与者
- 同一轮周期中逆势而亡的公司
- 曾经被视为"颠覆者"但最终失败的案例

#### 分析框架

| 维度 | 反面案例 | 成功企业 | 差异归因 |
|------|---------|---------|---------|
| 战略选择 | 做了什么关键决策导致失败 | 同时期做了什么不同选择 | 差异的关键在哪 |

| 时点判断 | 进入/退出/扩张的时间 | 时间选择为什么更优 | 时间差带来了什么 |
| 执行差异 | 产品/运营/组织差别 | 可验证的执行差距 | 执行差距的后果 |
| 外部环境 | 是否不可抗力 | 如何应对外部环境 | 环境是公平的吗 |

**把第一阶段深度分析的全部结论 + 第二阶段四步分析（背景→现状→推演→正反面案例）全数汇合到一起，做最终的闭合综合交叉验证。**

**0. 诚实第一：找不到就是找不到，不够就是不够**
不夸大、不编造、不修饰。搜索没结果就说没结果，数据不够就标注"数据缺口，此结论置信度低"，找不到来源的不要包装成"疑似""推测"塞进去。宁可报告粗糙、留白、有缺口，也不能用虚假的完整感欺骗读者。分析可以浅，不能说谎。

1. **质量优先于速度**：速度可以快，但不能以丢掉质量为代价。快速判断≠可以粗糙，深度模式≠可以潦草。每一级效率分级都有对应的交付底线——数据要有出处、逻辑要闭环、不确定要标注。宁可多花时间把一件事做透，也不要赶出一份需要返工的半成品。

2. **先判断行业类型再侦察**：不同类型走不同分支，不能套一套模板打天下
3. **先判断生命周期再侦察**：不盲目采集数据，先确定行业阶段和分析重点
4. **数据要有来源**：每个关键数字标注出处，AI估算的必须标注"估算"
5. **全球视角**：每个环节必须覆盖全球Top3，不能只看中国
6. **具体化**：不说"增长快"，说"3年CAGR XX%"；不说"壁垒高"，说"良率仅XX%，XX公司独家掌握XX工艺"
7. **写判断不写框架**：不教什么是渗透率什么是PEST，直接给数据和判断
8. **标注不确定性**：置信度低于60%的用"疑似""待验证"标记
9. **区分事实与判断**
10. **及时性**：搜索时优先用freshness参数获取最新数据
11. **砍废话**：不写框架解释、不写方法论说明、不凑字数
12. **多元视角**：每个关键结论至少从3个视角（投资者/经营者/技术演进/监管/客户）分别论证，视角冲突要明写
13. **连锁效应**：数据不是死的数字，每个关键变量必须展开"动了会怎样"的连锁传导——谁受益、谁受损、时间差多久
14. **说人话**：专业术语首次出现时，必须用一句大白话自然嵌入正文解释，不能堆术语不解释，也不能单独搞术语表。解释要融在句子里，让外行人也能顺读下去

# 📖 天眼使用手册

> **本手册是指导建议，不是教条。** 天眼是灵活的分析工具，不是固定流程模板。手册告诉你能做什么、怎么做，但具体怎么用由你自己判断。首次使用时自动发送，后续随时说"天眼使用手册"即可调取。

## 一、天眼是什么

行业侦察一体化引擎。输入一个行业名称，天眼帮你完成从行业定位、产业链拆解到竞争格局、投资判断的全流程分析。适合创业调研、投资研究、竞品分析、行业入门等场景。

## 二、怎么用

直接跟天眼说你想看什么行业：

| 你说 | 天眼做什么 |
|------|-----------|
| "XX行业怎么样" | 快速判断：生命周期+四维评分+一句话结论 |
| "侦察XX行业" | 标准侦察：全流程报告 |
| "深度分析XX行业" | 深度分析：完整产业链拆解+技术路线+竞争格局 |
| "拆解XX竞品/XX产业链" | 专项拆解：特定方向深度 |
| "无限/∞/往死里挖XX" | ∞无限制：深度分析+五步深化 |
| "对比XX和XX行业" | 行业对比：双行业独立分析+对比矩阵 |
| "XX品牌分析/XX商品为什么火" | 品牌与商品五段分析 |
| "XX公司融资/被谁投了" | 投资机构分析（26家数据库+五维评分） |

## 三、输出什么

- **快速判断** → 简短结论（.md）
- **标准及以上** → 全息报告（.md + .html可视化版）
- **深度/∞模式** → 派发子session并行执行，各模块独立产出后汇总
- 报告存放在 `./行业侦察/{行业名}/` 目录下

## 四、报告包含什么

1. **一句话定位**：这个行业的核心判断
2. **生命周期与四维判断**：导入/成长/成熟/衰退 + 可行性/规模性/防守性/盈利性
3. **产业链拆解**：每层做什么→全球Top3→中国玩家→壁垒→替代难度→话语权→数据
4. **技术路线交锋**：多路线对比+谁会赢+演进时间线
5. **全球竞争格局**：Top10+中国竞争力+非中国龙头
6. **市场与景气度**：规模/增速/订单/产能
7. **多元视角交叉**：投资者/经营者/技术/监管/客户
8. **估值与投资判断**：估值水平+核心风险

## 五、分析原则

- **每条数据有来源**，不编造
- **不确定的标注置信度**，不伪装确定
- **数据不够诚实标注缺口**，不用框架凑数
- **全球视角**，不只限中国
- **连锁效应**：每个关键变量展开"动了会怎样"
- **只说人话**：专业术语第一次出现时自然解释

## 六、天眼不能做什么

- 不做具体投资建议（"这个股该不该买"）
- 不编造非上市公司的财务数据（搜不到就标注"定性判断"）
- 不分析不知名的非上市小公司
- 不替代专业法律/医疗/财务咨询

## 七、常见问题

**Q：分析太长了，能简短点吗？**
A：说"快速判断"或"简单说一下XX行业"。

**Q：能分析非上市公司吗？**
A：知名非上市公司（如字节跳动、大疆）可以，搜不到财务数据时标注"基于公开信息的定性判断"。不知名小公司不做。

**Q：能用英文分析吗？**
A：可以，直接说英文行业名。

**Q：分析结果准吗？**
A：天眼基于公开信息和AI判断，不保证100%准确。关键决策建议多方验证。
